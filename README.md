#  PhonePe Transaction Data Analysis & Visualization

##  Project Overview

This project focuses on analyzing and visualizing digital transaction data from the **PhonePe Pulse GitHub repository**. The objective is to extract meaningful insights about transaction trends, user behavior, and regional adoption of digital payments across India.

The project involves data extraction, cleaning, exploratory data analysis (EDA), and visualization to understand patterns in transaction volume, value, and categories.

---

##  Business Objective

The goal of this project is to:

* Understand the growth of digital payments over time
* Identify top-performing states and regions
* Analyze transaction behavior across categories
* Provide data-driven insights for improving digital payment adoption

---

##  Dataset Source

The dataset is sourced from the official PhonePe Pulse GitHub repository:

🔗 https://github.com/PhonePe/pulse

##  Technologies Used

* Python
* Pandas & NumPy
* Matplotlib & Seaborn
* Plotly (for interactive visualizations)
* Google Colab

---

##  Project Workflow

1. **Data Extraction**

   * Cloned and accessed data from GitHub repository
   * Parsed JSON files

2. **Data Transformation**

   * Converted raw JSON data into structured DataFrames
   * Extracted relevant features like State, Year, Quarter, Type, Count, and Amount

3. **Data Cleaning**

   * Checked for missing values and duplicates
   * Ensured correct data types

4. **Exploratory Data Analysis (EDA)**

   * State-wise transaction analysis
   * Year-wise growth trends
   * Category-wise distribution
   * Quarter-wise patterns
   * Correlation and relationship analysis

5. **Data Visualization**

   * Created multiple charts including:

     * Bar charts
     * Line charts
     * Pie charts
     * Heatmaps
     * Scatter plots
     * Pair plots

---

##  Key Insights

* Digital payments show consistent growth over the years
* A few states contribute significantly to transaction volume
* Peer-to-peer transactions dominate usage
* Strong correlation exists between transaction count and amount
* Regional disparities exist in digital payment adoption
* Transaction behavior varies across categories and regions

---

##  Business Recommendations

* Focus on high-performing states for expansion and premium services
* Improve awareness and infrastructure in low-performing regions
* Enhance features in high-usage categories like peer-to-peer payments
* Use data-driven strategies to boost adoption and engagement

This project demonstrates how data analysis can be used to understand digital payment trends and user behavior. The insights derived can help businesses and policymakers make informed decisions to improve digital payment ecosystems and drive financial inclusion.
